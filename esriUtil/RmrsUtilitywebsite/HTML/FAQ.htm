Frequently Asked Questions

  1. Why do function models not display correctly?
  2. Why do some function model take a long time to display?
  3. Why does stratified random sampling take a long time to finish?
  4. What happens if I don't have SAS installed?
  5. How is the Raster Analysis menu different than Spatial Analyst?
  6. Do you plan on incorporating R into the statistical modeling?
  7. How do I submit comments or report bugs and errors?

Answers

  1. Function models use statistics from the input datasets and make a guess at the output statistics. This can cause the surface to display incorrectly. To fix this simply rerun statics using the calculate statistic button or save the function model as a conventional dataset using the save raster button.
  2. When displaying function models each cell must be processed. For most function datasets this is a relatively quick process but function dataset that are a composite of multiple functions can require additional time to display.
  3. Stratified random sampling can take a long time to finish when raster zones represent a small portion of the landscape.
  4. Currently, if you don't have SAS you will not be able to use the Statistical Modeling tools.
  5. On the surface there appears to be many similarities between the Raster Analysis menu and Spatial Analyst extension coupled with Model Builder. The primary difference is how data are processed (not to mention some of the analyses that can be performed). The tools within the Raster Analysis menu uses delayed processing and lazy reading techniques to process data. We combine these methodologies into a modeling framework called Function Modeling that significantly reduces storage space and processing time associated with raster analysis and modeling. Also RMRS Raster Utility is free as compared to Spatial Analyst.
  6. Yes, we plan on integrating R into the statistical modeling library.
  7. By clicking on the Report a Bug link.
